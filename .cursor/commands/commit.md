# 分析变更并创建提交

## 任务

分析当前 Git 变更，生成规范的提交信息并执行提交。

## 执行步骤

1. **检查变更状态**
   - 运行 `git status` 查看未跟踪和已修改的文件
   - 运行 `git diff` 查看未暂存的变更内容
   - 运行 `git diff --staged` 查看已暂存的变更内容

2. **分析变更内容**
   - 识别变更类型（新功能、修复、重构、文档、样式等）
   - 总结变更的目的和影响
   - 确定影响的模块或组件

3. **生成提交信息**
   
   使用以下格式：
   ```
   <类型>(<范围>): <简短描述>

   <详细说明（可选）>
   ```

   类型说明：
   - `feat`: 新功能
   - `fix`: 修复 bug
   - `refactor`: 重构代码（不改变功能）
   - `style`: 样式调整（不影响代码逻辑）
   - `docs`: 文档更新
   - `chore`: 构建或辅助工具变更
   - `perf`: 性能优化

4. **执行提交**
   - 将相关文件添加到暂存区：`git add <文件>`
   - 执行提交：`git commit -m "<提交信息>"`
   - 显示提交结果：`git log --oneline -1`

## 注意事项

- 不要提交敏感文件（.env、credentials 等）
- 提交信息使用中文描述
- 一次提交只做一件事，保持原子性
- 提交前确认变更内容正确
